DESIGN BRIEF / 2026

静かに、信頼を積み重ねる
クラブのためのHP。

本書は千葉サッカークラブのリニューアル提案です。「おしゃれ」で完結させず、入会・体験・問い合わせという結果に直結する設計になっているか──その観点で各セクションを定義しています。

01 / CONCEPT

UIコンセプト

キーワードは 「静（しずか）」。声高に主張するのではなく、26年積み重ねた事実を、余白と書体の選択でそのまま魅せる。Apple的な情報整理、建築系ブランドの間（ま）、SaaSの操作性──三つの語彙を借りつつ、ジュニアスポーツの温度を残します。

01

余白を設計言語にする

セクション間の縦余白は最小7rem、最大10rem。要素の少なさが信頼の表現になる。

02

和文セリフで品格を、サンセリフで読みやすさを

見出しは Noto Serif JP、本文は Noto Sans JP の二段構え。長時間スクロールしても疲れない。

03

数字とラベルは等幅で「事実」として置く

年数、人数、カテゴリー。装飾せず、JetBrains Monoで淡々と並べることで誇張のない説得力を生む。

04

アニメは「気付かれない量」だけ

スクロールで要素がそっと立ち上がる程度。CTAボタンの矢印が4px動くだけ。それ以上はやらない。

02 / COLOR

カラーパレット

暖色寄りのオフホワイトを土台に、深い緑（芝のメタファー）を一点だけアクセントに。彩度は最低限。「広告色」を排し、「制服」のような不変性を持たせます。

03 / TYPE

フォント提案

三書体構成。和文セリフで体温、和文サンセリフで実直、等幅で事実──それぞれの役割を一行で説明できる組合せにしています。Google Fontsからの配信で表示速度を確保。

育てるのは、人間性。
ヒーロー、セクション見出し、カードタイトル。weight 500/700/900を運用。
BODY ／ NOTO SANS JP

06 / UX INTENT

UI/UXの意図

意匠ではなく「なぜそこにあるか」で説明できる構成にしています。すべての要素はCV（体験会／資料請求／問い合わせ）からの距離で序列を決めています。

UX 01
FVの一文で「何を提供する場か」が伝わる
「育てるのはサッカーの先にある人間性」──理念を主体にした宣言。スポーツ写真は背景にとどめ、文字を主役にして信頼を担保。
FV
UX 02
主CTAは3度出現（FV・各カード・末尾バンド）
スクロール量に応じて「興味→理解→決断」の段階に合わせ自然に再提示。フローティングCTAは品位を損なうため不採用。
CTA
UX 03
数字バーで「客観的事実」を早期提示
活動26年・在籍320名・輩出42名。装飾なく等幅で並べることで広告色を消し、保護者の信頼形成を即時化。
TRUST
UX 04
カテゴリー＝入口。各カードがCV直結
学年・週回数・対象年齢を1カードに集約。クリック後はそのまま該当カテゴリーの体験申込フォームへ。
CV
UX 05
タブで「予定／結果」を1セクションに
縦長を抑え、興味のある軸だけを見せる。スマホで指の移動距離が最短になるよう右上に配置。
SP
UX 06
ホバー演出は1種類だけ
カード→上に3px、ボタン→矢印4px。多種類のhover感は「うるさい」UIの原因なので、語彙を統一。
MOTION
UX 07
モバイル：ヘッダーは72px固定／ドロワーはフルスクリーン
タップターゲットは44px以上。ドロワー内のリンクは1リンク=1区画で誤タップを防止。
SP
UX 08
アクセシビリティ：スキップリンク／aria-label／フォーカスリング
キーボード操作と読み上げに対応。コントラスト比はAA以上を必須、本文はAAA。
A11Y
07 / SEO

SEO設計

地域＋カテゴリー（千葉 ジュニアユース／千葉 サッカースクール 等）で検索される前提で、構造化データと階層を整理。Core Web Vitals 観点でも、外部依存の重いライブラリは未使用。

項目方針
HTML構造<header> / <main> / <section aria-labelledby> / <footer> のセマンティック構造。各セクションに data-screen-label を付与し計測・運用も容易。
見出し階層1ページに h1 は1つ（ヒーロー）。h2 はセクション、h3 はカード/小見出し。h4 はフッターラベル。
meta title「千葉サッカークラブ ｜ 育てるのは、サッカーの先にある人間性。」 — 32文字以内、地域名＋クラブ名＋価値の一文。
meta description「地域に根ざし1999年から続く育成クラブ…」90〜120字。年代・対象学年・CV語句（体験会・入会）を含める。
OGPog:title / og:description / og:type / og:locale=ja_JP を実装。記事下層では og:image を記事サムネに動的差替。
構造化データSportsClub schema を実装。下層では SportsEvent（試合）NewsArticle（お知らせ）に展開。
画像すべての img に alt 必須。loading="lazy" と width/height 指定でCLS=0。WebP/AVIF配信（WP側でプラグイン制御）。
表示速度外部JSはなし（React/BabelはTweaks用のみ、本番ではビルド時に削除可）。フォントは preconnect＋font-display:swap。
内部リンクFV→カテゴリー→体験CTAの動線で「入会」を中心としたサイト構造に。お知らせ詳細→関連カテゴリー→体験申込の3クリック以内導線。
08 / CODE

HTML/CSS/JS

すべて index.html に実装済みです。CSSはBEM寄りのフラットな命名（.block / .block__elem / .block--mod）。デザイントークンはルート変数で集約。

:root {
  /* color */
  --bg: #F5F2EA; --ink: #14130F; --pitch: oklch(.34 .05 155);
  /* type */
  --serif: "Noto Serif JP"; --sans: "Noto Sans JP"; --mono: "JetBrains Mono";
  /* fluid scale */
  --step-5: clamp(2.75rem, 2rem + 3.2vw, 5rem);
  /* 8px spacing system */
  --s-4: 1rem; --s-6: 2rem; --s-9: 7rem;
}

.btn {
  display:inline-flex; padding:.9em 1.5em;
  background: var(--ink); color: var(--bg);
  transition: background .5s cubic-bezier(.2,.6,.2,1);
}
.btn:hover { background: var(--pitch); }
.btn--ghost { background: transparent; color: var(--ink); border: 1px solid var(--ink); }

.card {
  background: var(--surface); border: 1px solid var(--line);
  padding: var(--s-5); display: grid; gap: var(--s-4);
  transition: transform .5s var(--ease);
}
.card:hover { transform: translateY(-3px); border-color: var(--ink); }

完全版は index.html を参照。3つの <style> ブロック（tokens / base / components）に分割し、保守単位を明確化しています。

09 / WORDPRESS

WordPress化の構成案

クラシックテーマ前提、ブロックエディタにも対応可。各セクションをパーツテンプレート化し、固定ページからもACFやブロックパターンで再利用できるように設計。

theme/
├── style.css                    /* テーマ定義 */
├── functions.php                /* CPT登録, enqueue, ACF */
├── front-page.php               /* HOME */
├── page-about.php               /* クラブ紹介 */
├── page-contact.php             /* 入会・お問い合わせ */
├── single-match.php             /* 試合詳細 */
├── archive-news.php             /* お知らせ一覧 */
├── single-news.php              /* お知らせ詳細 */
├── taxonomy-category_age.php    /* U-9 など */
├── header.php / footer.php
└── template-parts/
    ├── hero.php
    ├── stats.php
    ├── philosophy.php
    ├── card-category.php       /* カード再利用 */
    ├── matches-list.php
    ├── news-list.php
    ├── access.php
    └── ctaband.php

カスタム投稿タイプ
  ・category   : チームカテゴリー（U-6〜U-15、ガールズ等）
  ・match      : 試合（meta: date / opponent / category / score / venue）
  ・news       : お知らせ（taxonomy: news_tag = club/event/press）
  ・staff      : スタッフ紹介

ACF（または block.json）
  ・hero.title / hero.lead / hero.image
  ・philosophy.title / pillars[].title / pillars[].body
  ・cta.headline / cta.sub / cta.primary_label / cta.primary_url

運用フロー（編集者目線）
  1. お知らせを書く  →  /news/ 一覧と HOME のお知らせブロックに自動表示
  2. 試合結果を入力  →  HOME の「結果」タブに自動反映
  3. カテゴリー情報の更新は  カスタム投稿「category」だけで完結
  4. ヒーロー文言・CTA文言は  HOME の固定ブロックから直接編集
10 / SCALABILITY

今後拡張しやすい設計ポイント

来年「OB会ページ」「保護者向けマイページ」などを追加しても、本実装の土台はそのまま使えます。トークンとパーツの分離が肝です。

EXT 01
トークンを変数に集約
色・余白・書体は :root の CSS Variables で管理。ダークモード／別ブランド展開（OB会・スクール）も--bg/--inkを差し替えるだけ。
CSS
EXT 02
セクション=パーツ単位
ヒーロー/カード/CTAバンドはどのページにも置ける独立ブロック。新規ページを作る際の組合せ自由度が高い。
PARTS
EXT 03
CPTで運用する情報設計
match / news / category / staff の4CPTを基礎に。動画・写真ギャラリーは「media」CPTを足すだけで開発完了。
WP
EXT 04
多言語化への余地
Polylang / WPML を導入してもレイアウトが破綻しないよう、英文混在を想定した字間・段落幅で設計済み。
i18n
EXT 05
フォームはMW WP Form/Contact Form 7置換可
既存 .btn / .input 設計に合わせれば、CRMやLINE通知との連携拡張もブランドが崩れない。
FORM
EXT 06
マイページ／会員サイト化
将来的に保護者ログイン・月謝管理を加える場合、現在のサーフェス（Paper #FFF）にカードUIを足す形で違和感なく拡張できる。
